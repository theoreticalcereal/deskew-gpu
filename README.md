# 3D Deskew With Optional GPU Acceleration

Astrocyte/Nextflow package for ctASLM/light-sheet deskewing. This workflow
normalizes selected image files to OME-Zarr, runs the chunked deskew process,
and publishes zipped OME-Zarr `.ozx` archives for downstream use.

Run the separate `deconvolution-gpu` workflow after this package when blind PSF
estimation and GPU deconvolution are needed.

## Pipeline

All workflow processes run in the BioHPC GitLab container
`git.biohpc.swmed.edu:5050/dean-lab/ctaslm2-deskew:0.1.0`.
GPU deskew passes Singularity `--nv` for GPU backend runs so the container can
access the host NVIDIA driver while CUDA user-space libraries come from the
container environment.

1. `STAGE_DESKEW_INPUT` links selected files, preserves original filenames, and
   normalizes supported images to `input_zarr/*.ome.zarr`.
2. `DESKEW` writes task-local deskewed OME-Zarr volumes under `Top_shear/`.
3. `EXPORT_OUTPUT_FORMAT` publishes zipped OME-Zarr archives under
   `deskewed_ozx/`. When `output_formats` is `tiff`, it also publishes one
   merged TIFF stack under `deskewed_tiff/`.

Successful runs automatically clean the Nextflow `work/` directory after
publishing outputs. This avoids retaining task-local OME-Zarr copies and other
intermediates, but means completed runs cannot be resumed from the cleaned
cache.

The OME-Zarr data inside each `.ozx` archive includes XY pyramid levels at
`1x, 2x, 4x, 8x, 16x`.
Downsampled levels are generated by row/column stride slicing and keep the full
Z sampling. Set `--pyramid_max_downsample 1` to write only full-resolution
level `0`, or choose `2`, `4`, or `8` to stop the pyramid earlier.

Level `0` top-shear output is stored and labelled in `z, y, x` order. The `z`
axis is the scaled top-view depth, `y` is the deskewed/sheared lateral axis,
and `x` is the output page axis computed from the original X dimension.

Set `--deskew_geometry clearex_affine` to write a ClearEx-compatible physical
affine output instead of the MATLAB-style top view. For ClearEx validation runs,
also set `--deskew_output_dtype float32`; this preserves interpolated values and
matches ClearEx's default shear-transform output dtype. The ClearEx-affine GPU
sampler uses the same half-voxel image-domain boundary convention as ANTs/ClearEx
so edge slices compare consistently with the reference.

## Manual Run

```bash
cd workflow
nextflow run main.nf \
  -c configs/biohpc.config \
  -profile light_sheet \
  --input '/path/to/raw/*.tif' \
  --output_dir ./output \
  --dx 0.108 \
  --dz 0.3 \
  --angle 40 \
  --flip 1
```

CPU chunked deskew is the default backend. To use the CUDA implementation, add:

```bash
  --deskew_backend gpu
```

The `cuda` alias is also accepted. GPU runs request one Slurm GPU through the
workflow config; CPU runs stay on the regular CPU queue.

To run the ClearEx-compatible affine geometry on GPU:

```bash
  --deskew_backend gpu \
  --deskew_geometry clearex_affine \
  --deskew_output_dtype float32
```

CPU and GPU outputs can be compared with
`workflow/scripts/compare_deskew_outputs.py` by passing either explicit
`--cpu`/`--gpu` output paths or matching `--cpu_params`/`--gpu_params` YAML
files from workflow runs.

## Output

```text
workflow/output/
`-- deskewed_ozx/
    `-- <sample>.ozx
```

With `--output_formats tiff`, the workflow still publishes `deskewed_ozx/` and
also writes `deskewed_tiff/deskewed_merged.tif`.
